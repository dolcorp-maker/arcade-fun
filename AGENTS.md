# Project Memory

## Family And Mission
- This repository is a father-and-son project for building educational games for kids.
- The family context matters: the son is 8, the daughter is 5, and the project is about learning and dreaming together.
- Default product direction: playful, child-friendly, educational experiences first.

## Platform Constraints
- The site is hosted on GitHub Pages.
- Everything should run as a static site from this repository.
- Prefer browser-only solutions: HTML, CSS, JavaScript, JSON, localStorage, and Web APIs.
- Avoid backend requirements unless explicitly requested.

## Working Style For Future Sessions
- Start from the assumption that this project is for young children and family use.
- Favor simple controls, readable UI, low-friction setup, and safe content.
- Preserve GitHub Pages compatibility when adding features.
- When repository docs conflict with the actual code, trust the live code and update docs deliberately.

## Task Agent Pack
- The repository includes a dedicated task-agent pack at `vscode-agent-pack/`.
- Use it as local guidance when work clearly matches one of its roles:
  - `implementation-planner` for feature planning
  - `debug-triage` for debugging and root-cause work
  - `frontend-design` for UI and responsive design tasks
  - `code-reviewer` for review requests
- Supporting standards live in `vscode-agent-pack/docs/`.
- The pack is guidance inside this repo, not a runtime dependency.
